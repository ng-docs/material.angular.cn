安装 Angular

跨平台开发

使用 Angular 以一种方式构建应用程序，并复用你的代码和技能来构建适用于任何部署目标的应用程序。适用于 Web、移动 Web、原生移动和原生桌面。

速度和性能

在当今的 Web 平台上实现最高速度，并通过 Web Workers 和服务器端渲染进一步提升速度。Angular 使你能够控制可扩展性。通过在 RxJS、Immutable.js 或其他推送模型上构建数据模型来满足庞大的数据需求。

出色的工具

使用简单、声明式的模板快速构建功能。使用你自己的组件扩展模板语言，并使用大量现有的组件。几乎所有 IDE 和编辑器都提供即时的 Angular 特定帮助和反馈。所有这些都汇集在一起，使你能够专注于构建出色的应用程序，而不是试图让代码正常工作。

深受数百万用户喜爱

从原型到全球部署，Angular 提供了支持 Google 最大应用程序的生产力和可扩展基础设施。

什么是 Angular？

Angular 是一个平台，它使使用 Web 构建应用程序变得容易。Angular 结合了声明式模板、依赖注入、端到端工具和集成最佳实践来解决开发挑战。Angular 使开发人员能够构建在 Web、移动或桌面端运行的应用程序。

架构概述

Angular 是一个用于使用 HTML 和 TypeScript 构建客户端应用程序的平台和框架。Angular 本身是用 TypeScript 编写的。它将核心和可选功能实现为一组 TypeScript 库，你可以将这些库导入到你的应用程序中。

Angular 应用程序的基本构建块是 NgModules，它们为组件提供编译上下文。NgModules 将相关代码收集到功能集；Angular 应用程序由一组 NgModules 定义。应用程序始终至少有一个根模块，该模块启用引导，并且通常还有更多功能模块。

组件定义视图，视图是一组屏幕元素，Angular 可以根据你的程序逻辑和数据在这些元素之间进行选择和修改。每个应用程序至少有一个根组件。

组件使用服务，服务提供与视图没有直接关系的特定功能。服务提供者可以作为依赖项注入到组件中，使你的代码模块化、可重用且高效。

组件和服务都只是类，它们带有装饰器，这些装饰器标记它们的类型并提供元数据，告诉 Angular 如何使用它们。

组件类元数据将其与定义视图的模板关联起来。模板将普通 HTML 与 Angular 指令和绑定标记结合起来，使 Angular 能够在渲染 HTML 以供显示之前修改它。

服务类元数据提供了 Angular 需要的信息，以便通过依赖注入 (DI) 使其可供组件使用。

应用程序的组件通常定义许多视图，这些视图按层次结构排列。Angular 提供 Router 服务来帮助你定义视图之间的导航路径。路由器提供复杂的浏览器内导航功能。
