Material 3 主题

什么是 Material 3

Material 3 (M3) 是 Google 开源设计系统 Material Design 的最新版本。它是 Material 2 (M2) 的继任者，Angular Material 一直遵循该设计系统。

从 v17.2.0 开始，Angular Material 除了 M2 之外，还包含对 M3 样式的实验性支持。该团队计划在 M3 经过短暂的实验阶段后稳定其支持，以便收集有关设计和 API 的反馈。

如何在你的应用中使用 Material 3

M3 在 Angular Material 中实现为一组备用 Sass 主题，这些主题与许多应用中使用的相同 Angular Material 组件集相同。要在 Angular Material 中使用 M3，请使用 @angular/material-experimental 中的 define-theme 函数创建你的主题，而不是使用 @angular/material 中的 define-light-theme 或 define-dark-theme 来创建 M2 主题。

定义你的主题

API 的最简单用法 $theme: matx.define-theme() 定义了一个具有默认值的主题。 但是，与它的 M2 对应项一样， define-theme 允许你通过传递一个主题配置对象来配置 Angular Material 应用的三个主题维度：颜色、排版和密度。配置对象可以具有以下属性。

属性 描述
color [可选] 颜色选项的映射表。有关详细信息，请参阅 自定义你的颜色。
typography [可选] 颜色选项的映射表。有关详细信息，请参阅 自定义你的排版。
density [可选] 密度选项的映射表。有关详细信息，请参阅 自定义你的密度。 
@use '@angular/material-experimental' as matx;

$theme: matx.define-theme((
  color: (
    theme-type: dark,
    primary: matx.$m3-violet-palette,
  ),
  typography: (
    brand-family: 'Comic Sans',
    bold-weight: 900
  ),
  density: (
    scale: -1
  )
));

自定义你的颜色

可以通过主题配置对象的 color 属性自定义应用颜色的以下方面（请参阅 M3 颜色规范 了解有关这些术语的更多信息）：

颜色属性 描述
theme-type [可选] 指定主题类型， light 或 dark。
primary [可选] 指定应用主色调调色板使用的调色板。（注意：M3 规范中描述的次要、中性、中性变体调色板将根据你的主色调自动选择，以确保和谐的色彩组合）。
tertiary [可选] 指定应用三级色调调色板使用的调色板。

在 @angular/material-experimental 中有许多可用的调色板，可以与 primary 和 tertiary 选项一起使用：

  • $m3-red-palette
  • $m3-green-palette
  • $m3-blue-palette
  • $m3-yellow-palette
  • $m3-cyan-palette
  • $m3-magenta-palette
  • $m3-orange-palette
  • $m3-chartreuse-palette
  • $m3-azure-palette
  • $m3-violet-palette
  • $m3-rose-palette

自定义你的排版

可以通过主题配置对象的 typography 属性自定义应用排版的以下方面（有关这些术语的更多信息，请参阅 M3 排版规范）：

排版属性 描述
plain [可选] 用于普通文本（如正文）的字体系列。
brand [可选] 用于品牌文本（如标题）的字体系列。
bold [可选] 用于粗体文本的字体粗细。
medium [可选] 用于中等文本的字体粗细。
regular [可选] 用于常规文本的字体粗细。

自定义你的密度

可以通过主题配置对象的 density 属性自定义应用密度的以下方面：

密度属性 描述
scale [可选] 应用中组件的紧凑程度，从 0（最宽松）到 -5（最紧凑）。

应用你的主题

创建主题后，可以使用与应用 M2 主题相同的 -theme、 -color、 -typography、 -density 和 -base 混合器来应用它（有关这些混合器的更多信息，请阅读 通用主题指南）。对于 M3 主题，这些混合器 对发出的样式做出了一些保证。这些保证 不适用于传递 M2 主题。

  • 混合器在您指定的精确选择器下发出属性。它们 不会添加到 选择器或增加规则的特异性。
  • 混合器只发出 CSS 自定义属性声明 （例如 --some-prop: xyz）。它们 不会发出任何标准的 CSS 属性，如 color、 width 等。

使用组件颜色变体

许多组件都有一个 color 输入属性，允许开发人员应用组件的不同颜色变体。使用 M3 主题时，此输入仍然会向 组件添加一个 CSS 类（例如 .mat-accent）。但是，没有针对这些类的内置样式。你可以通过将 $color-variant 选项传递给组件的 -theme 或 -color 混合器来应用颜色变体。

<mat-checkbox class="tertiary-checkbox" />
<section class="tertiary-checkbox">
  <mat-checkbox />
</section>

@use '@angular/material' as mat;
@use '@angular/material-experimental' as matx;

$theme: matx.define-theme();

.tertiary-checkbox {
  @include mat.checkbox-color($theme, $color-variant: tertiary);
}

此 API 更灵活，并且生成更少的 CSS。例如，上面显示的 .tertiary-checkbox 类可以应用于任何复选框 或包含复选框的任何元素，以更改该元素中所有复选框的颜色。

虽然你应该优先使用带有颜色变体的混合器，但你也可以选择使用 提供的向后兼容混合器 ，它们直接将样式应用于现有的 CSS 类（ mat-primary、 mat-accent 和 mat-warn）。

使用 CSS 自定义属性进行细粒度自定义

主题混合器发出的 CSS 自定义属性源自 M3 的设计标记。要进一步 自定义超出 define-theme API 的 UI，可以在样式中手动设置这些自定义属性。

主题混合器做出的保证意味着你不需要针对组件的内部选择器或使用过高的特异性来覆盖任何这些标记化的属性。始终将你的基本主题应用于应用程序的根元素（通常是 html 或 body），并在适用的最高级别选择器上应用任何覆盖。

<mat-sidenav-container>
  Some content...
  <mat-sidenav>
    Some sidenav content...
    <mat-checkbox class="danger">Enable admin mode</mat-checkbox>
  </mat-sidenav>
</mat-sidenav-container>

@use '@angular/material' as mat;
@use '@angular/material-experimental' as matx;

$light-theme: matx.define-theme();
$dark-theme: matx.define-theme((
  color: (
    theme-type: dark
  )
));

html {
  // Apply the base theme at the root, so it will be inherited by the whole app.
  @include mat.all-component-themes($light-theme);
}

mat-sidenav {
  // Override the colors to create a dark sidenav.
  @include mat.all-component-colors($dark-theme);
}

.danger {
  // Override the checkbox hover state to indicate that this is a dangerous setting. No need to
  // target the internal selectors for the elements that use these variables.
  --mdc-checkbox-unselected-hover-state-layer-color: red;
  --mdc-checkbox-unselected-hover-icon-color: red;
}

使用 Material 3 主题对自己的组件进行主题化

用于读取 M2 主题属性的相同实用函数（在 对组件进行主题化的指南 中描述）可用于读取 M3 主题的属性。但是，根据规范，可用的命名调色板、排版级别等对于 M3 主题是不同的。

主题对象的结构被认为是实现细节。代码不应依赖于直接读取它的属性，例如使用 map.get。始终使用 Angular Material 提供的实用函数来访问主题的属性。

@use '@anuglar/material' as mat;
@use '@anuglar/material-experimental' as matx;

@mixin my-comp-theme($theme) {
  .my-comp {
    font: mat.get-theme-typography($theme, body-large, font);
    letter-spacing: mat.get-theme-typography($theme, body-large, letter-spacing);
    background: mat.get-theme-color($theme, surface);
    @if mat.get-theme-type($theme) == dark {
      color: mat.get-theme-color($theme, primary, 20);
    } @else {
      color: mat.get-theme-color($theme, primary, 80);
    }
    padding: 48px + (2px * mat.get-theme-density($theme));
  }
}

如何将应用从 Material 2 迁移到 Material 3

Angular Material 不提供从 M2 到 M3 的自动迁移，因为应用的设计是主观的。Material Design 提供了一般原则和约束来指导你，但最终如何将这些原则应用到你的应用中取决于你。也就是说，Angular Material 的 M3 主题在设计时就考虑了最大程度的兼容性。

更新使用 Angular Material 主题的组件，使其与 Material 3 兼容

为了便于从 M2 平滑过渡到 M3，你的组件可能需要同时支持 M2 和 M3 主题。一旦整个应用迁移完成，就可以移除对 M2 主题的支持。

最简单的方法是检查主题版本，并为 M2 和 M3 发射不同的样式。你可以使用 @angular/material 中的 get-theme-version 函数来检查主题版本。该函数将为 M2 主题返回 0，为 M3 主题返回 1（有关如何从 M3 主题中读取值的详细信息，请参阅 使用 Material 3 主题为自己的组件设置主题）。

@use '@angular/material' as mat;

@mixin my-comp-theme($theme) {
  @if (mat.get-theme-version($theme) == 1) {
    // Add your new M3 styles here.
  } @else {
    // Keep your old M2 styles here.
  }
}

在全局主题样式中传递一个新的 M3 主题

使用 define-theme 创建一个新的 M3 主题对象，并在之前传递 M2 主题的所有地方传递它。所有接受 M2 主题的 Angular Material 混合都与 M3 主题兼容。

更新不支持 Material 3 主题的 API 的用法

由于 Material 3 是作为 Material 2 使用的相同组件的备用主题实现的，因此两者的 API 在很大程度上是相同的。但是，有一些需要注意的差异：

  • M3 预计任何 @include 的 -theme、 -color、 -typography、 -density 或 -base 混合都应该包装在一个选择器中。如果你的应用在根级别包含这样的 @include，我们建议将其包装在 html { ... } 中。
  • M3 具有不同的 API 用于设置组件的颜色变体（有关更多信息，请参阅 使用组件颜色变体）。
  • <mat-tab-group> 的 backgroundColor 属性不受支持，不应与 M3 主题一起使用。
  • <mat-button-toggle> 的 appearance="legacy" 变体不受支持，不应与 M3 主题一起使用。
  • 对于 M3 主题，调用 all-component-typographies 不会 发射 typography-hierarchy 样式，因为这会违反 M3 不添加选择器的保证。相反，如果你想在你的应用中使用这些样式，则必须显式调用 typography-hierarchy 混合。
  • typography-hierarchy 混合输出的 CSS 类名对应于 M3 类型比例级别名称，而不是 M2 类型比例级别名称。如果你依赖 M2 类来为你的应用设置样式，则可能需要更新类名。

（可选）为颜色变体添加向后兼容样式

我们建议 不要依赖 M2 主题提供的许多 Angular Material 组件提供的 color="primary"、 color="accent" 或 color="warn" 选项。但是，如果你想快速更新到 M3，并且愿意接受为这些变体生成的额外 CSS，则可以启用恢复此 API 行为的向后兼容样式。从 @angular/material-experimental 中调用 color-variants-back-compat 混合，并传入要为其生成颜色变体样式的 M3 主题。

@use '@angular/material' as mat;
@use '@angular/material-experimental' as matx;

$theme: matx.define-theme();

html {
  @include mat.all-component-themes($theme);
  @include matx.color-variants-back-compat($theme);
}

（可选）为类型比例层次结构添加向后兼容样式

使用 M3 主题调用 typography-hierarchy 混合会为与 M3 类型比例名称匹配的 CSS 类名（例如 .mat-headline-large）生成样式，而不是与 M2 类型比例名称匹配的 CSS 类名（ .mat-headline-1）。如果你在你的应用中使用 M2 类名，我们建议将所有用法更新为新的类名之一。但是，为了使迁移更容易， typography-hierarchy 混合支持除了新的类名之外还发射旧的类名。我们尽力将 M2 类映射到 M3 中合理的等效类。要启用这些样式，请向混合传递一个额外的参数 $back-compat: true。

@use '@angular/material' as mat;
@use '@angular/material-experimental' as matx;

$theme: matx.define-theme();

@include mat.typography-hierarchy($theme, $back-compat: true);

FAQ

我可以使用除预定义的 Material 3 调色板以外的颜色吗？

目前，我们只提供预定义的调色板，但我们计划在使 M3 API 稳定并在 @angular/material 中可用时添加对使用自定义生成的调色板的支持。

我可以依赖 CSS 自定义属性名称保持稳定吗？

我们可能会在将 API 从实验性 API 中移出之前更改自定义属性名称，但这些更改将伴随着一个原理图，用于在你的整个应用中查找并替换旧名称为新名称。

Material 2 样式和 API 会消失吗？

Material 2 样式及其 API 将继续得到支持，我们目前没有立即弃用它们的计划。我们理解应用切换到最新的 Material 3 样式需要时间，我们希望提供足够的时间进行迁移。当我们决定移除这些 API 时，它们将被标记为已弃用，并在接下来的两个主要版本中继续得到支持。截至目前，它们不被视为已弃用。
